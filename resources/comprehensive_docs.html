1. Introduction

Welcome to the Docker Manager documentation. This comprehensive guide covers both the REST API backend and the React frontend web application.

The Docker Manager is a complete solution for managing Docker installations, services, containers, images, and Docker Compose projects through a web interface. It provides a RESTful API for programmatic access and a user-friendly web interface for interactive management.

Key Features

  • Full Docker Management - Install, configure, and manage Docker and Docker Compose
  • Container Operations - Start, stop, restart, and monitor containers
  • Image Management - Pull, build, tag, and remove Docker images
  • Docker Compose Support - Manage multi-container applications
  • System Information - Monitor Docker system resources and status
  • Web Interface - Modern React-based user interface
  • REST API - Complete programmatic access to all features
  • Real-time Updates - WebSocket support for live status updates

Technology Stack

Backend (C++)

  • Language: C++17
  • Framework: Crow (C++ microframework)
  • Build System: CMake
  • JSON Library: json11
  • Process Management: Custom implementation

Frontend (React)

  • Framework: React 18 with TypeScript
  • State Management: React Query
  • Routing: React Router
  • HTTP Client: Axios
  • UI Components: Custom component library
  • Styling: Tailwind CSS
  • Build Tool: Vite

2. REST API Documentation

The Docker Manager REST API provides programmatic access to all Docker management features. All endpoints are relative to the base URL.

Base URL

http://localhost:14040

2.1 Authentication

Some operations require sudo permissions. These endpoints require a valid sudo password to be provided in the request. If a sudo password is required but not provided or invalid, the API will return a 401 Unauthorized response.

POST /api/test/sudo

Tests and sets the sudo password.

Request Body:

{
  "password": "your_sudo_password"
}

Response:

{
  "success": true,
  "message": "Sudo access available"
}

export class DockerApiService {
  // Docker Information
  static async getDockerInfo(): Promise {

5. Installation and Setup

Prerequisites

  • Linux system (tested on Fedora 30)
  • C++17 compiler (GCC 9+)
  • CMake 3.10+
  • Node.js 16+
  • npm or yarn

Backend Setup

# Clone the repository
git clone 
cd container_installer_rest

# Build the project
mkdir build
cd build
cmake ..
make

# Run the application
./docker_manager

Frontend Setup

# Navigate to frontend directory
cd frontend

# Install dependencies
npm install

# Start development server
npm run dev

# Build for production
npm run build

API Access

Once the backend is running, the API will be available at:

http://localhost:14040

The web interface will be available at:

http://localhost:5173

API Documentation

Interactive API documentation is available at:

http://localhost:14040/api/docs
